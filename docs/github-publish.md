## First Release Notes

`oss-signal-kit` v0.1.0 provides a safe, dependency-light CLI that collects public GitHub repository signals and renders Markdown/JSON reports for maintainers.

Highlights:

- read-only GitHub API access
- no telemetry
- no AI provider calls
- no repository mutation
- Markdown and JSON output
- application draft snippets under 500 characters
- CI, tests, security policy, issue templates, and contribution guide

## Codex for Open Source Application Positioning

Use this repository as a maintainer productivity and OSS sustainability tool. The strongest framing is:

`oss-signal-kit helps maintainers turn scattered public repository signals into reviewable evidence packs for grant applications, sponsorship pages, project updates, and AI-assisted maintenance workflows. API credits would be used to add optional maintainer-reviewed summaries, documentation drift checks, issue triage assistance, and release note generation without allowing autonomous repository writes.`
